## Социальная сеть для путешественников

**Функциональные требования:**
- публикация постов из путешествий с фотографиями, небольшим описанием и привязкой к конкретному месту путешествия
- оценка и комментарии постов других путешественников
- подписка на других путешественников, чтобы следить за их активностью
- поиск популярных мест для путешествий и просмотр постов с этих мест
- просмотр ленты других путешественников и ленты пользователя, основанной на подписках в обратном хронологическом порядке.

**Нефункциональные требования:**
- DAU 15_000_000 (нужен запас прочности для роста аудитории)
- аудитория стран СНГ
- доступность 99,9 - 99, 95
- совместимость:
	- кросплатформенность
	- поддержка Google Chrome и Safari
	- адаптивный дизайн
	- PWA
- данные будут хранится всегда
- активность пользователей:
	- в среднем публикация 2 поста в неделю
	- в среднем публикация 5 комментариев и 200 лайков в день
	- в среднем 10 подписок в неделю
	- в среднем чтение 5 постов в день
	- в среднем чтение 100 комментариев в день
	- в среднем чтение ленты 5 раз в день по 20 постов
	- в среднем 2 поиска в неделю
- лимиты:
	- максимум 10 фотографий в посте
	- максимальный размер текста в посте 5_000 символов
	- максимум 200 постов в день
	- максимальная длина комментария 2_000 символов
	- максимум 200 комментариев/лайков в день
	- максимум 150 подписок в день
	- максимум 100 поисков в день
- сезонности нет
- тайминги:
	- публикация - 2 сек
	- поиск - 3 сек
	- загрузка ленты - 1 сек для 20 постов
- объем данных:
	- публикация (текст + фотографии) = 5_000 * 4B (UTF-8 буквы и смайлики) + 10 * 800Kb = 0,02Mb + 8Mb = 8,02Mb
	- комментарий (текст) = 2_000 * 4B (UTF-8 буквы и смайлики) = 8Kb
	- лайк (dest_user_id, post_id, user_id, created_at) = 128B
	- подписка (dest_user_id, user_id, created_at) = 96B
	- поисковая выдача (сжатое фото + 500 символов текста * 10 шт) = 1Mb
	- лента (сжатое фото + 500 символов текста * 20 шт) = (500Kb + (500 * 4B)) * 20 = 502,2Kb * 20 = 10Mb

**Оценка нагрузки:**
- **RPS write:**
    - RPS для публикации = 15_000_000 * 2 / 7 / 86_400 = 50
    - RPS для комментария = 15_000_000 * 5 / 86_400 = 870
    - RPS для лайка = 15_000_000 * 200 / 86_400 = 34_700
    - RPS для подписки = 15_000_000 * 10 / 7 / 86_400 = 250

- **RPS read:**
    - RPS для чтения публикации = 15_000_000 * 5 / 86_400 = 870
    - RPS для чтения комментариев = 15_000_000 * 100 / 86_400 = 17_400
    - RPS для поиска = 15_000_000 * 2 / 7 / 86_400 = 50
    - RPS для чтения ленты = 15_000_000 * 5 / 86_400 = 870

**Оценка трафика:**
- **Write:**
    - Публикация поста = 50 * 8,02Mb = 401 Mb/s
    - Публикация комментария = 870 * 8Kb = 7 Mb/s
    - Публикация лайка = 34_700 * 128B = 5 Mb/s
    - Подписка = 250 * 96B = 24 Kb/s

- **Read:**
    - Чтение публикации = 870 * 8,02Mb = 7 Gb/s
    - Чтение комментариев = 17_400 * 8Kb = 0,4 Gb/s
    - Поиск = 50 * 1Mb = 50Mb
    - Чтение ленты = 870 * 10Mb = 8,7Gb/s
